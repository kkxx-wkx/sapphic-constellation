# Sapphic Constellation｜拉拉星系图

Sapphic Constellation 是一个以隐私和同意为核心的关系星系图项目。它的灵感来自“把人与人之间的连接可视化成星系网络”的概念，但这个项目并不是一个公开八卦式关系图，而是一个强调匿名、同意、私密和用户控制权的网页应用。

在这个项目中，任何人都不会在未经同意的情况下出现在别人的关系图里。用户只能通过私密邀请链接邀请别人加入，对方打开链接后，可以选择匿名加入、使用显示名、隐藏连接，或拒绝邀请。

## 项目简介

这个网页应用允许用户创建一个属于自己的关系星系图。每一个人会被显示为一个节点，每一段经过双方同意的关系会被显示为节点之间的连接线。

项目的核心理念是：

> 没有明确同意，就不应该被展示在别人的关系图里。

用户可以生成私密邀请链接，并通过微信、QQ、小红书私信、iMessage 或其他聊天软件手动发送给对方。对方打开链接后，可以自主决定是否加入，以及以什么方式出现。

## 核心功能

### 1. 私密邀请链接

用户可以生成一个私密邀请链接，并手动发送给想邀请的人。

本项目没有使用自动邮件发送功能，而是采用复制链接的方式，更适合国内用户常见的社交软件使用场景。

### 2. 基于同意的加入流程

当对方打开邀请链接后，可以选择以下几种方式回应：

* 匿名加入
* 使用显示名加入
* 私密加入，但隐藏这条连接
* 拒绝邀请

只有在对方明确同意后，系统才会创建对应的人物节点和关系连接。

### 3. 匿名或显示名选择

用户可以选择以匿名方式出现，也可以填写一个自己愿意展示的显示名。这让用户可以控制自己想公开多少身份信息。

### 4. 交互式星系图

经过同意的人会以节点形式出现在星系图中，经过同意的关系会以连接线的形式展示。

目前支持的连接类型包括：

* 朋友
* 约会过
* 同一活动认识
* 朋友介绍

### 5. Supabase 数据存储

项目使用 Supabase 存储用户资料、人物节点、关系连接和邀请信息。

主要数据表包括：

* `profiles`
* `people`
* `connections`
* `invitations`

## 当前已知问题

当前用户自己的节点还没有稳定地作为中心节点显示在星系图中。

原因是：用户资料目前存储在 `profiles` 表中，而星系图页面主要从 `people` 表读取节点。因此后续需要确保用户创建或更新 profile 时，也会在 `people` 表中创建或更新一个代表自己的 self-node。

计划中的修复方式：

* 在 `people` 表中加入 `profile_id` 字段
* 用户创建或更新 profile 时，同步 upsert 一个自己的节点
* 设置 `owner_id = 当前用户 id`
* 设置 `profile_id = 当前用户 id`
* 在星系图中识别并高亮当前用户自己的节点

## 隐私原则

这个项目遵循以下隐私原则：

1. 同意优先。
2. 没有明确同意，任何人都不应该被展示。
3. 默认支持匿名参与。
4. 用户可以拒绝邀请。
5. 私密连接不应被公开展示。
6. 不做“谁最受欢迎”“谁连接最多”“谁约会最多”这类排名。
7. 项目应该用于个人关系回顾、社群连接理解和可视化表达，而不是曝光、比较或八卦。

## 本地运行方式

安装依赖：

```bash
npm install
```

启动本地开发服务器：

```bash
npm run dev
```

打开本地页面：

```text
http://localhost:5173/
```

## 环境变量

在项目根目录创建 `.env` 文件，并加入：

```env
VITE_SUPABASE_URL=你的 Supabase 项目地址
VITE_SUPABASE_ANON_KEY=你的 Supabase anon key
```
